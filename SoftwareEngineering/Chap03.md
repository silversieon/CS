>
1. 명세화, 설계 및 구현 프로세스가 중첩된다. ↔ (
2. 시스템을 점진적으로 구현한다. ↔ (
3. 공식적 미팅보다는 비공식적인 커뮤니케이션을 통해 문서화를 최소화한다. ↔ (

무거운 계획 주도 개발 접근법으로 시스템을 개발하면 오버헤드가 크기 때문에, 유연성이 좋은 애자일 기법을 사용한다.

> 애자일 기법 원칙
>
- 고객과의 협업
- 변화에 대응
- 프로세스가 아닌 사람 즉, 개인과의 상호작용을 중시
- 문서보다 작동하는 소프트웨어를 점진적으로 개발

## 애자일 개발 기법

> 익스트림 프로그래밍(XP)
>

품질 높은 소프트웨어를 고객과 빠르게 소통하며 유연하게 개발하는 프로세스

> XP 원칙과 실무
>
- 공동 소유권: 참여하는 개발자들이 코드에 대한 공동 책임을 지며, 어떤 것이든 변경할 수 있다
- 연속적 통합(CI): 특정 작업이 끝나면 전체 시스템에 통합 및 전체 테스트
- 점증적 계획: 사용자 스토리를 카드로 기록하고, 작업으로 쪼개어 진행
- 고객의 참여: 고객은 개발팀의 구성원이며, 시스템 요구사항을 전달할 책임이 있음
- 짝 프로그래밍: 개발자가 서로의 작업을 점검 및 지원하기 위해 짝을 이루어 작업
- 리팩토링: 코드 개선 사항이 발견되면 리팩토링하는 것.(변경이 아닌 같은 로직에 대해 유지보수)⭐
- 소규모 릴리스: 비즈니스 가치를 제공하는 최소한의 유용한 기능 먼저 개발
- 테스트 우선 개발: 기능 자체를 구현하기 전 테스트를 먼저 작성하는 것
- 칸반(Kanban): 작업 흐름을 보드위에 시각화하여 진행 상태를 표시, 병목을 줄이는 방식
- 단순한 설계(Simple Design): 현재의 요구사항만을 만족시키기 위한 충분한 설계를 하는 것⭐

> 사용자 스토리
>

일종의 시나리오로, 사용자가 요구사항을 제안하도록할 때 유용하게 사용한다.

애자일 기법에서는 요구 사항의 변경을 처리하기 위해서 별도의 요구공학 활동을 두지 않는다. 오히려 요구사항 도출과 개발을 함께 진행한다.

문제점 : 시스템의 중요한 요구사항 전체를 다룰 수 있을 정도로 충분한지 판단하기 어려움

> 리팩토링⭐
>

프로그래밍 팀이 개선 가능한 부분을 찾아서 즉시 구현하는 것을 의미한다.

익스트림 프로그래밍(애자일 기법)에서는 변경을 고려한 설계는 헛수고라는 생각을 기반에 둔다. 종종 예상한 변경이 발생하지 않거나, 전혀 다른 변경 요청이 이루어지기도 하기 때문이다.

점증적 개발의 근본적 문제는 부분적인 변경이 소프트웨어 구조를 망가뜨리는 것인데, 리팩토링은 이러한 문제를 막기 위해 사용한다.

> 테스트 우선 개발
>

테스트 주도 개발로 확장 된 테스트 우선 개발은 코드를 작성한 후에 코드에 대한 테스트를 만드는 것이 아닌,

코드를 작성하기 전에 테스트를 먼저 작성한다.

테스트 작성을 통해서 개발하는 기능에 대한 명세를 암묵적으로 정의, 이해할 수 있다.

> 짝 프로그래밍
>

프로그래머가 소프트웨어를 짝으로 개발하는 것.

장점

1. 시스템에 대한 집단 소유와 책임 개념 제공
2. 작성하는 모든 라인을 최소한 두 명이 살펴보기 때문에 비정형적 리뷰 프로세스를 진행하게 되며, 오류 발견에 효과적이다.
3. 리팩토링 사용

## 애자일 프로젝트 관리(스크럼 기법)

> 스크럼
>

짧은 개발 주기(스프린트)를 반복하며, 팀 협업과 빠른 피드백을 통해 제품을 점진적으로 완성해나가는 실행 방식

- 애자일 개발은 팀이 사용할 수 있는 시간과 자원을 최대한 잘 이용할 수 있도록 관리해야한다.
- 따라서 어느정도 진행 중인 내용에 대한 외부 가시화 제공을 위해 이 스크럼 방식을 사용한다.
- 스크럼은 애자일 프로젝트 조직에게 프레임워크를 제공하는 데 초점을 맞추며, 특정 개발 방법의 사용을 강요하지 않는다.

> 스크럼 용어
>
- 제품 백로그: 스크럼 팀이 해결해야 할 일에 대한 목록
- 스크럼: 전체 팀이 서로 직접 만나서 진행하는 짧은 일일 회의
- 스크럼 마스터: 팀이 스크럼을 효과적으로 활용할 수 있도록 보장하며, 외부 요인에 의해 방해받지 않도록 함
- 스프린트: 짧은 주기의 개발을 의미하며 주로 2~4주 길이(끝내지 못 한 작업 처리를 위해 늘려선 안 됨)
- 스크럼 보드: 스프린트 백로그, 완료 작업, 팀원 상황 등 회사 칠판(공유 자원)
