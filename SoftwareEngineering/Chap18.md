# 📌18장 서비스 지향 소프트웨어 공학

### SOA(Service Oriented Architecture)

실행 가능한 서비스를 표준 인터페이스를 통해 느슨하게 결합해 시스템을 구성하는 아키텍처 스타일

서비스는 **독립적**이며 **재사용 가능하며,** 구현 기술과 무관하게 **인터페이스로만 통신**

> 구성 예시
>
1. 서비스 제공자: 서비스 설계 및 구현, 인터페이스 명세, 레지스트리에 등록
2. 서비스 요청자: 표준 서비스 프로토콜을 이용해 서비스 명세 검색을 통해 제공자를 찾음
3. 서비스 레지스트리

- SOAP(Simple Object Access Protocol): SOA에서 서비스간 메시지를 교환하기 위한 표준 프로토콜
- WSDL(Web Service Description Language): SOAP 기반 웹 서비스의 인터페이스를 정의하는 명세 언어(XML 기반)

### RESTful 서비스

Representational State Transfer의 약자로 가벼운 방식의 웹 서비스 아키텍처

기본적인 요소인 자원으로 하여금 자원의 전송표현을 기반으로 한다.

RESTful 아키텍처에서는 모든 것이 자원으로 표현되며, 자원들은 URL처럼 각기 다른 식별자가 있다.

웹은 RESTful 아키텍처 시스템이며, 웹페이지가 자원이며, 식별자는 URL이다.

웹 프로토콜 http와 https는 네 가지 동작에 기반하며, 자원은 오퍼레이션에 대응된다.

RESTful 서비스와 SOAP 기반 서비스의 중요한 차이는 RESTful 서비스가 XML 기반이 아니라는 점.

json 기반으로 서비스 호출에 포함한 오버헤드 타임을 줄일 수 있다.
