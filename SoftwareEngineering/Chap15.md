# 📌15장 소프트웨어 재사용

재사용의 이점: 신속한 개발, 높은 품질, 비용 절감, 리스크 감소 등

재사용 단위: 시스템, 어플리케이션, 컴포넌트, 객체와 함수

개념 재사용: 코드 재사용 대신 소프트웨어의 기본적인 아이디어를 재사용

### 재사용 관점 용어

- 어플리케이션 프레임워크: 어플리케이션 생성을 위한 추상, 구체 클래스의 집합
- 아키텍처 패턴: 시스템 전체 구조를 설계하기 위한 반복적으로 검증된 상위 수준 설계 방식
- 디자인 패턴: 객체나 클래스 간의 상호작용에서 자주 발생하는 문제에 대한 재사용 가능한 설계 방식
- 프로그램 라이브러리: 특정 기능을 수행하는 재사용 가능한 함수 또는 클래스 모음으로 개발자가 호출하여 사용
- 기존 시스템 포장: 기존 시스템을 수정하지 않고 인터페이스를 감싸 재사용하거나 통합하는 기법
- 관점 지향 소프트웨어 개발(AOSD): 공통 관심사를 핵심 로직과 분리하여 모듈화하는 개발 방법
- 컴포넌트 기반 소프트웨어 공학(CBSE): 독립적이고 재사용 가능한 컴포넌트를 조립해 시스템을 구축하는 공학 기법
- 서비스 지향 시스템(SOA): 네트워크 기반의 독립적인 서비스를 결합하여 기능을 제공하는 시스템 구조

### 어플리케이션 프레임워크

1. 유사한 형태의 어플리케이션에서 사용될 수 있는 일반 기능을 제공
2. 추상 클래스와 구체 클래스의 집합으로 구현됨
3. 어플리케이션을 위한 핵심 아키텍처 제공

ex) GUI: 이벤트 처리 기능, 위젯 집합 기능 등 / WAF: 웹 어플리케이션 프레임워크(Spring)

시스템 구현을 위해서 일반적 프레임워크를 확장하여 구체적인 어플리케이션을 생성한다.

- 추상클래스를 상속 받아 구체클래스 구현
- 이벤트가 일어나면 호출되는 콜백 메서드 구현
