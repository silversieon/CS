# 📌 5장 : 시스템 모델링

## 시스템 모델링

> 시스템의 추상 모델을 개발하는 프로세스, 각 모델은 시스템의 서로 다른 뷰나 관점을 나타냄
>

- 요구공학 프로세스에서는 시스템 상세 요구사항을 이끌어내기 위해 사용
- 설계 프로세스에서는 구현할 엔지니어들에게 시스템을 설명하기 위해 사용
- 구현 후에는 시스템의 구조와 동작을 문서화하기 위해 사용

## 다이어그램

어떤 시스템 모델이 해당 다이어그램을 사용하면서 시각화 하는지 알아두기

### 액티비티 다이어그램

> 프로세스나 데이터 처리와 관련된 액티비티들을 제어 흐름과 함께 보여줌
>

- 컨텍스트 모델
    - 여러 자동화된 시스템을 포함하는 환경을 보여주는 모델
- 동작 모델(데이터 주도 모델링)
    - 입력 데이터의 처리와 이와 연관된 출력 생성과 관련된 일련의 행동
    - 최초 입력 처리와 이에 따른 출력부터 시스템 반응을 연속적으로 보여줌

### 유스케이스 다이어그램

> 사용자가 어떤 인터페이스를 통하여 시스템에 무엇을 기대하는지, 사용자(또는 시스템)와 시스템 사이의 외부 상호작용을 보여줌
>

- 상호작용 모델
- 화살촉 없는 선을 사용하는게 정석
- 화살표를 사용한다면 의존하는쪽이 화살표를 받는다

### 시퀀스 다이어그램**⭐**

> 액터와 시스템의 객체들 간의 상호작용과 객체들 간의 상호작용을 모델링하여 보여줌
>

- 상호작용 모델(시스템 상호작용)
- 동작 모델 (시스템 처리 순서)
- 관련된 객체들과 액터들은 다이어그램 상단에 나열, 수직으로 점선이 그어짐
- 상호작용들은 위에서 아래로 시간 순서로 배치

### 클래스 다이어그램**⭐⭐**

> 클래스들과 그들 간의 연관을 보여줌
>

- 구조 모델
    - 시스템을 구성하는 컴포넌트들과 그들 간의 관계
    - 클래스 다이어그램은 정적 구조 모델
- 각 클래스는 그와 연결된 클래스에 대해 어느 정도 알고 있어야한다.

### 상태 다이어그램**⭐**

> 시스템의 상태와 한 상태에서 다른 상태로 전이를 일으키는 이벤트를 보여줌
>

- 이벤트 주도 모델
    - 어떻게 시스템이 외부와 내부 이벤트에 반응하는지
    - 시스템이 유한한 수의 상태를 가지고 이벤트가 상태 전이를 일으킨다는 가정
